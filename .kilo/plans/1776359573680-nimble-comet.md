## 2. Stuck Tabbed Panels — Close (X) & Reconnect

**Problem**: When SSH connection drops or phone screen cycles, session tabs become stuck — can't be closed or reused. The `isConnected` flag goes `false` (via `shell.done` callback at `ssh_provider.dart:141-145`), but the tab remains with no way to close or reconnect.

**Changes**:

### `lib/screens/home_screen.dart` — `_buildSessionTabBar`
- Replace `ChoiceChip` with a custom tab widget that includes:
  - Session name (same as before)
  - **X close button** on each tab (calls `ssh.removeSession(s.id)`)
  - Visual indicator: green dot when connected, red dot when disconnected

### `lib/screens/home_screen.dart` — `ClientTab.build`
- When `active.isConnected` is `false`, instead of just showing static "Not connected" text, show:
  - "Session disconnected" message
  - **Reconnect button** — calls `ssh.connectSession(active.id)` (re-uses the existing profile from the session entry)
  - **Close button** — calls `ssh.removeSession(active.id)`

### `lib/providers/ssh_provider.dart` — `disconnectSession`
- After disconnecting, do NOT replace `terminal` with a fresh `Terminal()`. Instead keep the existing terminal so the user can see the last output and choose to reconnect or close.

## 3. SFTP Drive Switching (Windows)

**Problem**: SFTP browser defaults to `currentPath = '.'` which resolves to CWD (typically `C:\`). On Windows, users can't navigate to `D:\` or other drives because `..` only goes up to the drive root, not across drives.

**Changes**:

### `lib/widgets/sftp_browser.dart`
- Add a drive selector row at the top (before the path display)
- On first load, detect available drives by trying `listdir` on common Windows drive paths (`C:\`, `D:\`, `E:\`, etc.) or by parsing the result of listing the root
- Add a `List<String> availableDrives` state field
- Add a `_detectDrives()` method that tries listing `C:/`, `D:/`, `E:/`, etc. (up to Z:) and keeps the ones that succeed
- Add a dropdown/chip row for selecting the drive, which sets `currentPath` to the drive root and refreshes
- Initialize `currentPath` to `/` instead of `.` for consistency, or detect the initial CWD

### `lib/services/sftp_helper.dart`
- Add `listDrives()` method that tries to list common Windows drive roots and returns successful ones
- Alternatively, list the root `/` and look for drive-like entries (on Windows SSH servers, drives may appear as `C:`, `D:`, etc. in the root listing)

### Implementation approach for drive detection:
1. Try `sftp.listdir('/')` — on Windows SSH servers (e.g., OpenSSH), drives often appear as `C:`, `D:` entries in the root
2. If root listing contains entries matching pattern `[A-Z]:`, treat them as drives
3. If root listing doesn't reveal drives, try probing `C:/`, `D:/`, etc. individually
4. Show drives as a chip row at the top of the SFTP browser for quick switching
