### Testing

### Code Patterns Observed
- Uses Material Design widgets

## Plan for AGENTS.md Content

1. **Commands Section**
   - `flutter pub get` - Install dependencies
   - `flutter build apk` / `flutter build ios` / `flutter build windows` - Build for platforms
   - `flutter analyze` - Run linter
   - `flutter test` - Run all tests
   - `flutter test test/widget_test.dart` - Run single test file
   - `flutter test --name "test name"` - Run single test by name

2. **Code Style Guidelines**
   - Follow Flutter/Dart conventions
   - Use analysis_options.yaml rules
   - Import conventions (package: prefix)
   - Naming: camelCase for variables/methods, PascalCase for classes/types
   - Error handling: try-catch with proper error types
   - Use const constructors where possible

3. **Project Structure**
   - `lib/` - Application code
   - `lib/screens/` - Screen widgets
   - `lib/widgets/` - Reusable widgets
   - `lib/providers/` - State management
   - `test/` - Test files
